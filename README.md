# BIS Standards Recommendation Engine

> AI-powered RAG system for Indian MSEs to discover relevant Bureau of Indian Standards (BIS) regulations from product descriptions.

---

## Architecture Overview

```
PDF (BIS SP 21, ~900 pages)
        ↓
  [pdfplumber] → Extract text + tables
        ↓
  [Structure parser] → Detect IS XXXX blocks
        ↓
  [Metadata enricher] → category, year, keywords
        ↓
  ┌─────────────────────────────┐
  │      Index (built once)     │
  │  FAISS (BGE-large)          │
  │  BM25 (rank-bm25)           │
  └─────────────────────────────┘
        ↓ (query time)
  Hybrid Retrieval (Dense + Sparse)
        ↓
  Reciprocal Rank Fusion (RRF)
        ↓
  Cross-encoder Reranker
        ↓
  Rationale Generator (chunk-grounded)
        ↓
  Hallucination Validator (whitelist check)
        ↓
  Strict JSON Output
```

**Key Design Decisions:**
- **One chunk = one BIS standard** → maximizes retrieval precision
- **Hybrid search** = BM25 catches exact IS codes; BGE captures semantic meaning
- **Whitelist validation** = zero hallucinations guaranteed (all standard IDs verified against PDF)
- **Fast mode (no LLM)** = rationale extracted directly from chunk text → latency < 1s

## Setup

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

> **GPU users**: Replace `faiss-cpu` with `faiss-gpu` in requirements.txt for faster indexing.

### 2. Build the index (run once)

```bash
python setup_pipeline.py --pdf data/dataset.pdf
```

This will:
- Parse all ~900 pages (takes 5–15 minutes)
- Create `data/chunks.json` (structured standard blocks)
- Create `data/index/` (FAISS + BM25 indices)

**Model downloaded automatically**: `BAAI/bge-large-en-v1.5` (~1.3GB)

### 3. Run inference

```bash
python inference.py --input data/public_test_set.json --output data/results.json
```

## Input / Output Format

**Input** (`--input`):
```json
[
  {"id": "q1", "query": "Ordinary portland cement for RCC construction"},
  {"id": "q2", "query": "TMT steel bars for earthquake-resistant buildings"}
]
```

**Output** (`--output`, strict format):
```json
[
  {
    "id": "q1",
    "retrieved_standards": ["IS 269", "IS 12269", "IS 455"],
    "latency_seconds": 0.83
  },
  {
    "id": "q2",
    "retrieved_standards": ["IS 1786", "IS 432", "IS 2062"],
    "latency_seconds": 0.91
  }
]
```

## Hallucination Prevention

Three layers of protection:
1. **Whitelist**: Every IS standard ID extracted from PDF at index time. No ID outside this set can appear in output.
2. **Prompt constraint**: LLM instructed to only reference standards in the provided context.
3. **Post-processing**: Regex scan of all output standard IDs verified against whitelist before JSON serialization.

## Latency Tuning

| Mode | Avg Latency | Quality |
|---|---|---|
| Fast (no LLM) | ~0.5–1s | Good (chunk extraction) |
| LLM (TinyLlama) | ~2–3s | Better |
| LLM (Llama 3.1 8B GGUF) | ~3–4s | Best |

Default is **fast mode** to safely stay under 5s on all hardware.
